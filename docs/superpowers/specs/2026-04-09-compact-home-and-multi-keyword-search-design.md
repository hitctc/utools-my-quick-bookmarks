# 首页压缩、多关键词搜索与键盘引导设计

## 1. 背景

当前首页已经具备：

- 基于 Chrome 书签解析结果的卡片展示
- uTools 顶部输入框搜索
- 上下键切换高亮、回车打开
- 置顶、最近打开、打开次数、本地主题模式

但现状仍有四个明显问题：

- 首页首屏说明区过高，挤占了真正用于展示书签的垂直空间
- 卡片信息层级过厚，不适合 uTools 小窗口场景
- 搜索仍是单字符串包含匹配，不支持多关键词分词和结果高亮
- 首页缺少足够直接的键盘操作提示，用户容易忽略“上下键选择”的入口

这轮目标不是扩展数据能力，而是围绕“小窗口内更高效地看、搜、选、开书签”继续收紧首页体验。

## 2. 已确认约束

### 2.1 视觉与模式

- 以浅色模式作为这轮主设计基准
- 深色模式做等价适配，不改变交互结构
- 沿用 Nothing 风格的单色、机械感、信息密度设计

### 2.2 字体

按 `nothing-design` 约束，这轮继续使用以下字体名作为本地优先字体栈：

- `Space Grotesk`
- `Space Mono`
- `Doto`

受当前仓库“不要新增运行时远程字体请求”的约束，这轮不在 `index.html` 新增 Google Fonts `<link>`，继续使用本地优先字体栈和现有 fallback。

### 2.3 搜索与键盘

- 多关键词搜索采用“空格分词 + 全部命中”
- 关键词需要在首页卡片中高亮
- 用户使用上下键切换时，当前高亮书签必须始终保持在可见区域
- 不全局拦截左右键，避免影响 uTools 顶部输入框里的光标移动
- 首页底部提供常驻简短提示，明确“上下键选择，回车打开”

## 3. 本轮要实现

- 首页首屏大说明区压缩为薄状态条
- 原首页中的介绍性文字、路径说明等迁移到设置页
- 首页卡片极限压缩，保留真正高频的信息
- 顶部输入框搜索支持多关键词分词与全部命中
- 标题、域名、目录路径支持命中高亮
- 如果命中来自隐藏的完整 URL，补一个“网址命中”提示
- 首页底部增加常驻操作提示，提醒用户使用上下键选择
- 设置页与返回首页入口改成 Nothing 风格 icon 按钮
- 设置按钮改为首页右上角悬浮，不额外占用布局高度

## 4. 本轮明确不做

- 不新增页面内搜索框
- 不把完整 URL 重新常驻展示回首页卡片
- 不恢复首页的大段说明文案
- 不让左右键也参与书签切换
- 不全局拦截左右键来弹 tip，避免打断顶部输入框原生编辑体验
- 不增加多浏览器、多 profile 或写回 Chrome 原书签文件能力

## 5. 首页结构重排

## 5.1 结构目标

首页只承担三件事：

- 看结果
- 选择结果
- 打开结果

因此首页要删除所有不直接服务于这三件事的高占位内容。

## 5.2 新首页结构

首页重排为以下结构：

1. 右上角悬浮设置 icon 按钮
2. 顶部薄状态条
3. 状态卡或结果区块
4. 底部轻量操作提示条

其中：

- 右上角按钮悬浮显示，不占额外文档流高度
- 顶部薄状态条只展示最关键的当前状态
- 结果区块保留 `置顶 / 最近打开 / 全部书签 / 搜索结果`
- 底部提示条负责操作引导，不承担瞬时 tip 的职责

## 5.3 首页保留的信息

首页允许保留的信息压缩为：

- 书签总数
- 当前主题状态
- 当前是否处于搜索中
- 当前分区和书签结果

首页不再保留：

- 当前书签文件长路径的大块展示
- “我的快捷书签”这类大标题说明块
- 使用说明整段文案
- 来源说明整段文案

## 6. 设置页信息迁移

原首页上这些内容迁移到设置页：

- 书签文件路径展示
- Chrome 默认路径的说明
- 使用顶部输入框搜索、上下键切换、回车打开的操作说明
- 主题模式说明
- 插件当前只支持 `macOS + Chrome Default profile` 的限制说明

设置页要承担“配置 + 说明”的职责，首页只承担“操作结果”的职责。

## 7. 卡片压缩设计

## 7.1 卡片目标

卡片要在 uTools 小窗口下仍能一屏显示更多结果，因此优先级是：

- 先保证信息密度
- 再保证识别效率
- 最后才保留装饰性内容

## 7.2 首页卡片保留字段

首页卡片只保留以下字段：

- 标题
- 域名
- 目录路径
- 置顶状态
- 打开次数（仅在设置允许且次数大于 0 时显示）

其中：

- 域名单独一行，超出尾部省略
- 目录路径单独一行，超出尾部省略
- 完整 URL 不常驻展示
- `READY / 可开` 这类无信息增量的状态完全删除

## 7.3 卡片建议结构

推荐使用更扁平的三层结构：

1. 顶部：标题 + 极小化 pin 按钮
2. 中部：域名一行、目录路径一行
3. 底部：轻量状态标签，例如 `已置顶`、`打开 3`、`网址命中`

卡片内不再保留：

- 高占位头像块，如果保留也必须显著缩小
- 完整 URL 常驻文本
- 来源块的大面积信息
- 占空间但无用的“可开”状态标签

## 7.4 卡片尺寸原则

卡片从“展示型大卡”改为“信息型紧凑卡”：

- 优先缩短垂直高度，而不是继续压缩文字字号
- 保持标题为最强视觉层级
- 域名和目录路径都属于次级信息
- 状态标签属于边缘信息，放在卡片底部或角落

最终表现应接近“极紧凑的技术清单卡”，而不是宣传式封面卡。

## 8. 搜索设计

## 8.1 分词规则

搜索输入来自 uTools 顶部输入框。

分词规则：

- 对输入先做 `trim`
- 按一个或多个空白字符切分
- 过滤空字符串
- 统一转成小写匹配

例如：

- `github issue`
- `react hook form`

都视为多个独立关键词。

## 8.2 匹配规则

每个关键词都必须命中同一卡片，卡片才会进入结果集。

命中字段包括：

- 标题
- 完整 URL
- 目录路径
- 域名

这意味着搜索采用 `AND` 语义，不是 `OR`。

## 8.3 高亮规则

首页卡片需要对可见文本中的命中词做高亮：

- 标题命中时，高亮标题中的对应片段
- 域名命中时，高亮域名中的对应片段
- 目录路径命中时，高亮目录路径中的对应片段

如果关键词只命中了完整 URL 中当前未展示的部分，则：

- 不额外展开完整 URL
- 只补一个小型命中提示：`网址命中`

## 8.4 搜索状态文案

首页的状态提示统一改成简短中文，例如：

- `搜索中`
- `待搜索`
- `未找到结果`
- `结果 12`

避免继续出现冗长英文提示句。

## 9. 键盘交互设计

## 9.1 上下键

- `ArrowDown`：选中下一项
- `ArrowUp`：选中上一项
- 当前高亮项切换后必须自动滚动到可见区域

滚动规则以“当前卡片完整或尽量完整出现在视口内”为准，不要求固定滚到顶部。

## 9.2 回车

- `Enter`：打开当前高亮卡片

打开后继续沿用现有最近打开和打开次数更新逻辑。

## 9.3 左右键

左右键不改变选中项，也不做全局拦截。

原因是当前 `uTools` 只提供 `subInputFocus()` / `subInputBlur()` 之类控制方法，没有可靠的“顶部输入框当前是否聚焦”读取接口。强行拦截左右键，会误伤用户在输入框里移动光标的原生体验。

因此这轮采用更保守且稳定的方案：

- 左右键保持默认行为
- 首页底部常驻提示明确“上下键选择，回车打开；多个关键词用空格分开”
- 交互引导依赖静态提示，而不是瞬时 tip

## 10. 文案压缩策略

这轮把首页高频看到的重点提示词尽量改成简短中文。

建议文案映射：

- `SETTINGS` -> `设置`
- `SYSTEM` -> `跟随`
- `DARK` -> `深色`
- `LIGHT` -> `浅色`
- `BOOKMARK FILE` -> `书签文件`
- `QUICK BOOKMARKS` -> `快捷书签`
- `PIN` -> `置顶`
- `UNPIN` -> `取消置顶`
- `OPEN 3` -> `打开 3`
- `SEARCH READY` -> `待搜索`
- `FILTERING` -> `搜索中`

原则是：

- 首页优先短中文
- 设置页可以保留更完整的中文说明
- 只有真正更准确的技术名词才保留英文

## 11. Nothing 风格按钮设计

## 11.1 设置按钮

首页设置按钮改为右上角悬浮的 Nothing 风格 icon 按钮：

- 线性图标
- 单色描边
- 轻边框
- 无阴影
- 悬浮不占布局高度

它应该更接近“系统控制键”，而不是一枚大胶囊文案按钮。

## 11.2 返回首页按钮

设置页的“返回首页”也改成同一套 icon 语言：

- 使用返回箭头或类似技术感的单线图标
- 与设置按钮保持相同尺寸体系和边框语言
- 允许搭配简短中文文字，也允许做成纯 icon + tooltip，但应优先保证直观

- 深色模式结构与交互保持一致

## 14. 结论

这轮优化的本质不是继续“做更多功能”，而是把当前已有能力压缩成更符合 uTools 小窗口场景的可操作首页。

设计中心保持为：

- 首页只展示最值得看的内容
- 搜索只返回更精准的结果
- 键盘交互对用户直觉更友好
- 所有说明性内容回收到设置页
- Nothing 风格继续保留，但优先服务密度与效率
