# 书签卡片空间键盘导航设计

## 1. 背景

当前书签插件已经支持通过 uTools 顶部输入框搜索书签，并用方向键切换高亮卡片。现有键盘逻辑主要是线性导航：

- `ArrowDown` 选择下一个可见卡片
- `ArrowUp` 选择上一个可见卡片
- `Enter` 打开当前高亮卡片
- `Escape` 聚焦回顶部搜索输入

这个逻辑在搜索后会遇到两个问题：

- 用户按 `ArrowLeft` / `ArrowRight` 时，uTools 顶部输入框会移动光标，而不是选择书签卡片。
- 现有 `ArrowUp` / `ArrowDown` 只是线性前后移动，不符合卡片网格的视觉方向。

这轮目标是把首页搜索结果和书签列表的方向键行为改成常规卡片网格导航：方向键始终控制当前可见书签卡片，高亮移动遵循页面上的实际空间位置。

## 2. 目标

### 2.1 用户体验目标

- 首页有可见书签卡片时，`ArrowLeft`、`ArrowRight`、`ArrowUp`、`ArrowDown` 都用于选择卡片。
- 搜索输入框中的光标不再响应左右方向键移动，避免和卡片导航冲突。
- 方向键选择逻辑符合用户对网格的直觉：
  - 左右优先在当前行移动。
  - 上下按当前卡片位置选择上一行或下一行最接近的卡片。
  - 到达边界时不做跳跃式循环。
- 允许跨分区移动。`置顶`、`最近打开`、`全部书签` 在键盘导航上被视为一个连续的可见卡片集合。

### 2.2 工程目标

- 保留现有 `visibleEntries` 线性顺序，继续用于回车打开、搜索结果排序和跨行左右移动。
- 将空间导航算法收口为纯函数，避免把几何计算散落在 `App.vue` 里。
- `App.vue` 只负责收集当前可见卡片 DOM 的几何信息，并把结果交给导航 helper。
- 为核心方向键规则补充单测。

## 3. 非目标

- 不新增页面内搜索框。
- 不改变搜索匹配规则、拼音搜索规则或高亮命中规则。
- 不改变书签排序、分区生成、置顶、最近打开、打开次数逻辑。
- 不做方向键循环导航。第一张再按左、最后一张再按右、最上行再按上、最下行再按下都保持不动。
- 不改写 Chrome 原始 `Bookmarks` 文件。

## 4. 导航规则

### 4.1 全局拦截边界

当满足以下条件时，首页拦截四个方向键并阻止默认行为：

- 当前视图是 `home`
- 已完成初始化
- 当前不是阻塞加载状态
- 首页没有阻塞错误
- 当前存在至少一张可见书签卡片
- 用户没有同时按下 `Meta`、`Ctrl` 或 `Alt`

被拦截的按键：

- `ArrowLeft`
- `ArrowRight`
- `ArrowUp`
- `ArrowDown`

拦截后必须调用 `event.preventDefault()`，让 uTools 顶部输入框不再移动光标。

### 4.2 左右导航

`ArrowRight`：

- 如果当前行右侧存在卡片，选择右侧最近的卡片。
- 如果当前行右侧不存在卡片，但线性顺序里还有下一张卡片，选择下一张卡片。
- 如果当前卡片已经是最后一张可见卡片，保持不动。

`ArrowLeft`：

- 如果当前行左侧存在卡片，选择左侧最近的卡片。
- 如果当前行左侧不存在卡片，但线性顺序里还有上一张卡片，选择上一张卡片。
- 如果当前卡片已经是第一张可见卡片，保持不动。

这意味着：

- 当前卡片在一行最右侧时，按右可以进入下一行第一张。
- 当前卡片在一行最左侧时，按左可以回到上一行最后一张。
- 这种跨行左右移动沿用 `visibleEntries` 的线性顺序。

### 4.3 上下导航

`ArrowDown`：

- 找到当前卡片下方最近一行中的候选卡片。
- 在该行候选中选择横向中心点距离当前卡片中心点最近的卡片。
- 如果下方没有任何卡片，保持不动。

`ArrowUp`：

- 找到当前卡片上方最近一行中的候选卡片。
- 在该行候选中选择横向中心点距离当前卡片中心点最近的卡片。
- 如果上方没有任何卡片，保持不动。

上下导航不使用线性前后移动。它只按真实空间位置找上一行或下一行。

### 4.4 跨分区规则

导航候选来自当前页面所有可见书签卡片，不按区块隔离。

示例：

- `置顶` 区最后一行下方是 `最近打开` 区卡片时，按下可以进入 `最近打开` 区。
- `最近打开` 区最后一张右侧没有卡片，但线性顺序下一张是 `全部书签` 区第一张时，按右可以进入 `全部书签` 区。
- 区块标题、说明文字、状态条、设置按钮不参与导航。

## 5. 实现设计

### 5.1 DOM 标记

每张可见卡片所在容器增加稳定 DOM 标记：

- `data-bookmark-card-key="<entry.cardKey>"`

`entry.cardKey` 继续来自现有 `buildSectionEntries`，保证同一轮渲染内稳定、唯一。

### 5.2 几何信息结构

`App.vue` 在方向键事件中读取所有可见卡片 DOM，并转换为轻量结构：

```ts
type KeyboardNavigationRect = {
  cardKey: string
  index: number
  left: number
  right: number
  top: number
  bottom: number
  centerX: number
  centerY: number
}
```

`index` 必须对应 `visibleEntries` 的线性顺序。这样左右跨行时可以安全回退到上一张或下一张。

### 5.3 纯函数拆分

保留现有 `getKeyboardNavigationResult` 的职责：

- 判断当前按键是否应该被首页处理。
- 判断是否需要 `preventDefault()`。
- 判断 `Enter`、`Escape` 的行为。

新增纯函数：

```ts
getSpatialNavigationIndex({
  key,
  highlightedIndex,
  rects,
}): number
```

职责：

- 只处理 `ArrowLeft`、`ArrowRight`、`ArrowUp`、`ArrowDown`。
- 根据卡片几何位置返回下一个高亮索引。
- 找不到有效目标时返回原 `highlightedIndex`。
- 不直接访问 DOM，不依赖 Vue。

`App.vue` 的方向键流程：

1. 调用 `getKeyboardNavigationResult` 判断是否接管按键。
2. 如果是方向键移动，收集当前可见卡片 rects。
3. 调用 `getSpatialNavigationIndex` 得到目标索引。
4. 更新 `highlightedIndex`。
5. 继续复用现有的滚动保持可见逻辑。

### 5.4 行识别

因为当前 CSS 使用：

```css
grid-template-columns: repeat(auto-fill, minmax(220px, 1fr));
```

列数会随窗口宽度、滚动条和响应式布局变化，所以不通过 CSS 估算列数。

行识别直接基于 DOM 几何位置：

- 同一行判断以 `centerY` 接近为准。
- 上一行 / 下一行按 `centerY` 小于或大于当前卡片中心点筛选。
- 最近一行取纵向距离最小的一组候选。
- 同一目标行里再按 `centerX` 距离选择最接近的卡片。

实现时允许设置一个小的像素容差，用于吸收浏览器布局中的亚像素差异。

## 6. 边界情况

- 没有可见卡片：方向键不接管，保持当前行为。
- 当前高亮索引超过可见卡片数量：沿用现有 watcher，把索引压回安全范围。
- DOM rect 数量和 `visibleEntries` 数量不一致：不执行空间移动，返回原索引，避免误选。
- 当前卡片 DOM 不存在：返回原索引。
- 用户按 `Meta` / `Ctrl` / `Alt` 加方向键：不接管，保留系统快捷键或输入框原行为。
- 设置页不接管方向键。
- 阻塞加载或首页阻塞错误时不接管方向键。

## 7. 测试设计

### 7.1 单测

更新 `tests/preload/keyboardNavigation.test.mjs`：

- `ArrowLeft` / `ArrowRight` 会被识别为移动行为并 `preventDefault()`。
- 方向键在非首页、加载中、错误态、无卡片时不接管。

新增或扩展空间导航测试：

- 同一行 `ArrowRight` 选择右侧最近卡片。
- 同一行 `ArrowLeft` 选择左侧最近卡片。
- 行尾 `ArrowRight` 选择线性下一张卡片。
- 行首 `ArrowLeft` 选择线性上一张卡片。
- 最后一张按右保持不动。
- 第一张按左保持不动。
- `ArrowDown` 选择下一行横向中心最接近的卡片。
- `ArrowUp` 选择上一行横向中心最接近的卡片。
- 最上行按上保持不动。
- 最下行按下保持不动。
- rect 数量异常时保持不动。

### 7.2 构建验证

改动会影响前端运行时和测试文件，至少执行：

```bash
npm test
npm run build
```

### 7.3 手动 smoke

在 uTools 开发模式中验证：

1. 进入插件首页。
2. 在顶部输入框输入关键词，让页面出现多张结果卡片。
3. 按 `ArrowLeft` / `ArrowRight`，确认输入框光标不移动，高亮卡片会左右移动。
4. 在一行最右侧按 `ArrowRight`，确认进入下一行第一张。
5. 在一行最左侧按 `ArrowLeft`，确认回到上一行最后一张。
6. 按 `ArrowUp` / `ArrowDown`，确认高亮按视觉上下行移动。
7. 在最上行按上、最下行按下、第一张按左、最后一张按右，确认保持不动。
8. 在置顶、最近打开、全部书签之间验证可以跨分区移动。
9. 按 `Enter`，确认打开当前高亮卡片。

## 8. 交付边界

本轮交付完成后，用户应能在搜索后只用方向键完成结果选择，不再遇到左右方向键移动输入框光标的问题。卡片选择行为以真实视觉网格为准，并允许跨分区连续移动。
