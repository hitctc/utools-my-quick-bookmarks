# Nothing 风格主题重构与三态主题设计

## 1. 背景

当前书签插件已经具备：

- Chrome 书签读取与解析
- 首页卡片展示
- 顶部输入搜索
- 插件内置顶、最近打开、打开次数
- 设置页路径配置和展示开关

但当前界面仍然停留在“通用浅色卡片 UI”阶段，主要问题是：

- 视觉层级不稳定，首页没有真正的主视觉焦点
- 卡片、区块、按钮大量依赖渐变、阴影和玻璃感，整体偏散
- 首页和设置页之间缺少统一的视觉语言
- 没有主题能力，无法同时支撑深色和浅色体验

这轮目标不是增加业务能力，而是把现有书签工具重做成一套更统一、更克制、更有辨识度的 `Nothing` 风格界面，并补齐 `跟随系统 / 深色 / 浅色` 三态主题设置。

## 2. 这轮要实现

### 2.1 视觉目标

- 首页和设置页统一切到 `Nothing` 风格的黑白工业感界面
- 深色和浅色都作为一等主题处理，不做“浅色派生深色”或“深色派生浅色”
- 通过字体、留白、边框、标签和单点 accent 建立层级，而不是继续依赖彩色背景和大阴影

### 2.2 交互目标

- 设置页新增 `主题模式` 分段控件：
  - `跟随系统`
  - `深色`
  - `浅色`
- 主题模式保存后即时生效
- 首页高亮、置顶、错误、搜索状态在深浅主题下都保持清晰可辨

### 2.3 工程目标

- 主题配置并入现有 `BookmarkUiSettings`
- 不新增新的本地存储 key
- 尽量限制在现有页面和样式结构内重做，不引入新框架或大规模重构

## 3. 这轮明确不做

- 不新增浏览器支持范围，仍然只按当前项目支持 `macOS + Chrome Default profile`
- 不增加页面内搜索框，搜索仍由 uTools 顶部输入框承担
- 不改写 Chrome 原 `Bookmarks` 文件
- 不扩展书签编辑、删除、分组管理、重复检测
- 不引入 UI 组件库、状态库或路由系统

## 4. 字体与设计系统

### 4.1 字体声明

本轮设计固定使用以下 Google Fonts：

- `Space Grotesk`
  - 用于正文、标题、说明文案、页面主体
- `Space Mono`
  - 用于按钮、标签、路径、统计数值、状态文字
- `Doto`
  - 只用于首页 hero 级数字，不用于正文

加载方式：

- 通过全局样式引入 Google Fonts
- 不假设本地系统已安装这些字体

### 4.2 视觉原则

- 主视觉只保留一个：`书签总数`
- 字体层级固定为三层：
  - 第一层：hero 数字或页面主标题
  - 第二层：区块标题、卡片标题、说明文本
  - 第三层：路径、来源、目录、状态标签
- 颜色以单色灰阶为主，只允许一处红色 accent 用于：
  - 当前高亮提示
  - 错误状态
  - 活跃状态提示
- 不使用大面积渐变、厚阴影、玻璃模糊

### 4.3 主题基调

- 深色主题：
  - 背景接近 `OLED black`
  - 卡片与面板通过深浅黑灰区分层级
  - 正文高对比，标签偏灰
- 浅色主题：
  - 背景接近技术手册纸张质感的暖白或冷白
  - 卡片为白色或浅灰色面板
  - 通过边框和留白分层，不靠阴影

## 5. 首页设计

## 5.1 首页三层信息结构

首页只保留三层重点：

### 第一层

- 左侧 hero 数字：当前书签总数
- 插件名称：`Quick Bookmarks`

### 第二层

- 当前书签文件路径摘要
- 当前搜索状态
- 主题模式状态
- 设置入口

### 第三层

- 各区块描述
- 卡片来源
- 目录路径
- 打开次数
- 置顶状态

## 5.2 首页布局

首页分为四块：

1. Hero 区
   - 左侧放超大数字和标题
   - 右侧放路径摘要、主题状态和设置按钮
2. 状态条
   - 用于显示搜索提示、加载、错误、空态
   - 文案更接近设备状态风格，例如：
     - `[ SEARCH READY ]`
     - `[ FILTERING ]`
     - `[ ERROR: ... ]`
3. 内容区块
   - `置顶`
   - `最近打开`
   - `全部书签`
4. 卡片网格
   - 保留当前网格结构，但卡片视觉整体重做

## 5.3 首页区块

区块不再使用柔和浮层卡片，而是改成技术面板式容器：

- 使用细边框和更克制的圆角
- 计数展示改为标签或计数器样式
- 区块标题和说明明确区分
- 通过留白和对齐表达分组关系

## 6. 书签卡片设计

## 6.1 卡片视觉

当前彩色渐变封面改为单色面板卡片：

- 不再使用 URL hash 生成的彩色渐变背景
- 不再依赖阴影和浮起感
- 采用黑白灰面板 + 细边框 + 局部 accent

## 6.2 卡片信息结构

书签卡片改成更稳定的信息布局：

1. 顶部
   - 来源 root
   - 域名
   - 置顶按钮
2. 中部
   - 标题
   - URL 摘要
3. 底部
   - 目录路径
   - 打开次数
   - 置顶状态标签

## 6.3 高亮与交互

- hover 和 focus 只改变：
  - 边框
  - 文本亮度
  - 背景轻微反差
- 不做 scale 放大和投影增强
- 当前键盘高亮项使用：
  - 更强边框
  - 一处红色点或红线
- 置顶按钮改为更机械的文字按钮或细线按钮，不再做悬浮胶囊

## 7. 设置页设计

## 7.1 设置页结构

设置页延续当前单页表单结构，但重做为技术手册式布局：

1. 顶部导航
   - 返回首页
   - `Settings` 标签
2. 主题设置区
   - 分段控件：`跟随系统 / 深色 / 浅色`
3. 路径设置区
   - 书签文件路径输入框
   - 保存并读取
   - 恢复默认路径
   - 刷新书签
4. 首页显示设置区
   - 首页显示最近打开
   - 显示打开次数

## 7.2 设置组件风格

- 表单标签统一使用 `Space Mono` 全大写
- 输入框偏技术表单风格，强调边框与对齐
- 开关和分段控件遵守 `Nothing` 风格：
  - 选中态明显
  - 非选中态克制
- 错误信息使用内联文本：
  - `[ ERROR: 当前书签文件路径不存在或不可访问 ]`

## 8. 主题模式设计

## 8.1 主题模式类型

新增主题模式字段：

```ts
type BookmarkThemeMode = 'system' | 'dark' | 'light'
```

并入现有设置：

```ts
interface BookmarkUiSettings {
  showRecentOpened: boolean
  showOpenCount: boolean
  themeMode: BookmarkThemeMode
}
```

## 8.2 生效规则

- `system`
  - 通过 `window.matchMedia('(prefers-color-scheme: dark)')` 计算当前有效主题
- `dark`
  - 强制深色
- `light`
  - 强制浅色

## 8.3 应用方式

- 根节点增加主题标记，例如：
  - `data-theme="dark"`
  - `data-theme="light"`
- 全局样式通过 CSS 变量切换主题 token
- 首页和设置页统一从同一套 token 读值

## 8.4 状态持久化

- 继续使用当前 UI 设置的存储链路
- 不新增新的 storage key
- 主题和其他展示开关一起归一化、读取、保存

7. 打开设置页制造错误路径，确认错误文案在深浅主题下都清晰

## 12. 风险与控制

### 12.1 主要风险

- 深浅主题 token 切换后，局部对比度不够
- 当前卡片和状态条改动较大，容易让键盘高亮态不明显
- `system` 模式如果只在初始化时读取，不监听系统变化，会表现不完整

### 12.2 控制策略

- 所有核心文本都必须在深浅主题下保持足够对比
- 高亮态优先保证可辨识，不追求极度克制到看不见
- `system` 模式实现为“初始化读取 + 监听变化”

## 13. 验收标准

满足以下条件即可认为这轮设计实现完成：

- 首页和设置页整体视觉已经切到明确的 `Nothing` 风格语言
- 深色和浅色都可用，且设置页可切换 `跟随系统 / 深色 / 浅色`
- 主题设置持久化，重进插件仍生效
- 书签卡片、区块、按钮、状态提示不再依赖旧的渐变玻璃风格
- 搜索、高亮、错误、置顶、打开次数在深浅主题下都清晰可用
