# 卡片展示、顶部输入搜索与插件内状态设计

## 1. 背景

当前项目已经具备：

- macOS 下 Google Chrome 默认书签文件路径读取
- 设置页修改书签文件路径
- `Bookmarks` JSON 解析
- 首页基础列表展示

接下来这一轮不再扩展“读书签文件”本身，而是把首页做成更像真正可用的书签工具：

- 首页书签展示样式迁移参考 `bookmarks-tab-cards`
- 搜索不再用页面内输入框，而是接入 uTools 顶部自带输入框
- 首页支持方向键切换和回车打开
- 增加插件内置顶、取消置顶、最近打开、打开次数、手动刷新

## 2. 用户要求

### 2.1 这轮要实现

- 卡片式展示
- 点击卡片打开书签
- 搜索书签
  - 使用 uTools 自带顶部输入框
  - 不在页面里额外增加一个搜索框
  - 模糊匹配 `标题 + URL + 文件夹路径`
  - 输入时实时筛选
  - 支持方向键选择
  - 回车直接打开当前高亮书签
- 插件内置顶
- 插件内取消置顶
- 手动刷新
  - 在设置页中以参考项目的方式提供入口
- 打开书签次数
  - 在设置页中提供显示开关
- 最近打开
  - 在设置页中提供首页显示开关

### 2.2 这轮明确不做

- 不做任何改写 Chrome 原 `Bookmarks` 文件的能力
- 不做改名称、改地址、改位置
- 不做删除重复书签、合并重复书签
- 不做重复书签检查
- 不改当前设置页的整体视觉风格

## 4. 搜索设计

## 4.1 搜索来源

搜索输入来自 uTools 顶部自带输入框，不在页面里新增独立搜索框。

实现上采用 `utools.setSubInput`：

- 插件进入后，注册顶部输入框监听
- 用户在顶部输入框输入时，页面内实时同步搜索词
- 页面根据搜索词即时筛选书签卡片

这样能保持“搜索框是 uTools 自带的，不是页面自己画的”。

## 4.2 匹配范围

搜索匹配以下字段：

- 书签标题
- 书签 URL
- 书签文件夹路径

第一阶段使用简单模糊匹配，不做拼音和分词扩展。

## 4.3 搜索结果表现

- 非空搜索词时，首页进入“搜索模式”
- 搜索模式下只展示匹配结果卡片
- 顶部保留标题和设置按钮，不额外显示页面内搜索框
- 无结果时显示明确空态

## 4.4 键盘交互

- `ArrowDown`：移动到下一个结果
- `ArrowUp`：移动到上一个结果
- `Enter`：打开当前高亮卡片对应的 URL

高亮项必须有明显视觉态，不依赖鼠标 hover。

## 5. 打开书签设计

打开书签统一通过 `utools.shellOpenExternal(url)`，交给系统默认浏览器处理。

每次打开时同时更新：

- 最近打开时间
- 打开次数

点击方式包括两种：

- 鼠标点击卡片
- 键盘回车打开当前高亮项

## 6. 插件内状态设计

这些状态全部只存在插件本地，不写回 Chrome 原书签文件。

### 6.1 置顶

- 只影响插件首页展示顺序
- 不改变 Chrome 书签真实顺序
- 置顶状态保存在 `utools.dbStorage`

### 6.2 最近打开

- 记录每个书签最后一次打开时间
- 首页可选展示“最近打开”区块
- 是否显示由设置项控制

### 6.3 打开次数

- 每次打开书签时累加
- 卡片上是否显示打开次数由设置项控制

### 6.4 手动刷新

- 设置页提供“刷新书签”按钮
- 点击后重新读取当前路径对应的 `Bookmarks` 文件
- 不做自动监听 Chrome 书签变化

## 7. 设置项设计

设置页样式保持当前项目现状，不迁移参考项目的抽屉视觉。

这轮新增的设置项：

```ts
interface BookmarkUiSettings {
  chromeBookmarksPath: string
  showRecentOpened: boolean
  showOpenCount: boolean
}
```

设置页需要支持：

- 修改书签文件路径
- 恢复默认路径
- 重新读取书签
- 首页显示最近打开：开 / 关
- 显示打开次数：开 / 关

## 8. 首页结构设计

设置页视觉保持不动，首页重做。

首页结构：

1. 顶部栏

2. 状态区

## 12. 排序规则

### 12.1 普通模式

- 置顶区：只显示置顶书签
- 最近打开区：按 `openedAt` 倒序
- 全部书签区：按原始解析顺序展示；置顶区内已单独展示的书签可不在全部区重复显示，也可以在全部区保留但弱化，二选一时推荐“不重复显示”

### 12.2 搜索模式

- 只显示匹配结果
- 优先按是否置顶、匹配强度、原始顺序综合排序
- 第一阶段可以先简化为：
  - 置顶优先
  - 其余保持原始顺序

## 13. 错误处理

- 搜索为空：回到普通首页模式
- 搜索无结果：显示空结果提示
- 打开 URL 失败：给出提示，不中断页面
- 本地状态存储失败：给出提示，但不影响原书签数据
- 刷新失败：保留当前页面，显示错误信息
